## Codebase Patterns

### Database Pattern (SQLite + better-sqlite3)
- Use `?` positional parameters for queries (NOT `$1` which is PostgreSQL syntax)
- Database is initialized lazily on first `getDb()` call
- Database file location: `data/task-queue.db` (gitignored)
- Use `closeDb()` to close connection in tests for clean state
- Delete database file between tests for isolation

### Testing Pattern
- Use vitest with globals enabled
- Test database: delete and recreate for each test
- Use `beforeEach` to clean up and `afterEach` to close connections

### Project Structure
- `lib/db.ts` — Database connection and types
- `migrations/*.sql` — Schema migrations
- `__tests__/*.test.ts` — Test files
- `app/api/*` — Next.js API routes (App Router)

### Frontend Patterns
- Use Lucide React for SVG icons (never emoji)
- Use CSS custom properties from globals.css for theming
- All clickable elements need `cursor-pointer` class
- Add hover states with 150-200ms transitions
- Use `focus-visible` ring for accessibility
- Dashboard uses client-side data fetching with auto-refresh
